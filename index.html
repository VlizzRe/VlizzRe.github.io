Motor a Pasos Bipolar

Tu guía completa para entender y usar motores a pasos bipolares

¿Qué es un Motor a Pasos Bipolar?

Un motor a pasos (o stepper motor) es un tipo de motor DC sin escobillas que divide una rotación completa en un número de pasos discretos. La posición del motor puede ser controlada con precisión sin ningún mecanismo de retroalimentación, siempre y cuando el motor esté dimensionado correctamente para la aplicación.

Los motores a pasos bipolares se distinguen por tener **dos bobinas** (o dos pares de bobinas). A diferencia de los motores unipolares que tienen una toma central en cada bobina, los bipolares requieren que la corriente fluya en ambas direcciones a través de sus bobinas, lo que significa que necesitan un controlador de puente H.

Un motor a pasos bipolar típico y su diagrama de conexiones.

Características Principales

  • **Mayor Torque:** Generalmente, los motores bipolares ofrecen un mayor torque para un tamaño dado en comparación con sus contrapartes unipolares, debido a un uso más eficiente de las bobinas.
  • **Cuatro Cables:** Un motor a pasos bipolar suele tener 4 cables de conexión (dos para cada bobina).
  • **Control Más Complejo:** Requieren un controlador de puente H (como un L298N, DRV8825, A4988) para invertir la dirección de la corriente en las bobinas.
  • **Precisión:** Son excelentes para aplicaciones que requieren movimientos precisos y repetibles, como impresoras 3D, CNC, robótica y actuadores de posicionamiento.

¿Cómo Funciona?

El movimiento de un motor a pasos bipolar se logra energizando secuencialmente sus dos bobinas en una secuencia específica para crear un campo magnético giratorio.

Secuencia de Pasos (Ejemplo de Paso Completo)

Asumiendo dos bobinas (A y B) y un controlador de puente H que puede energizar las bobinas en polaridad positiva (+) o negativa (-):

  1. **Paso 1:** Bobina A (+), Bobina B (apagada)
  2. **Paso 2:** Bobina A (+), Bobina B (+)
  3. **Paso 3:** Bobina A (apagada), Bobina B (+)
  4. **Paso 4:** Bobina A (-), Bobina B (+)
  5. **Paso 5:** Bobina A (-), Bobina B (apagada)
  6. ... y así sucesivamente, cada paso mueve el rotor una fracción de vuelta.

Existen diferentes modos de paso (paso completo, medio paso, micro paso) que ofrecen diferentes niveles de suavidad y precisión en el movimiento.

Conexión y Control con Arduino (Ejemplo Básico)

Para controlar un motor a pasos bipolar con Arduino, necesitarás un controlador de motor. El **DRV8825** o el **A4988** son muy populares por su facilidad de uso.

Conexión Básica:

  • **Motor a Driver:** Conecta los dos pares de bobinas del motor a las salidas A y B del driver (ej. 1A/1B y 2A/2B para un L298N; o A1/A2 y B1/B2 para DRV8825).
  • **Driver a Arduino:**
    • Pin STEP del driver a un pin digital de Arduino (para enviar pulsos de paso).
    • Pin DIR del driver a otro pin digital de Arduino (para controlar la dirección).
    • ENABLE del driver (si existe) a HIGH o a GND según el driver para habilitarlo.
    • Alimentación del motor a la entrada de alimentación del driver.
    • Arduino y driver deben compartir la misma GND.

Código Arduino (Pseudocódigo):


#include <Stepper.h> // o la librería para tu driver (ej. AccelStepper)

const int stepsPerRevolution = 200; // Pasos por revolución de tu motor
const int dirPin = 2; // Pin de dirección del driver
const int stepPin = 3; // Pin de paso del driver

void setup() {
  pinMode(dirPin, OUTPUT);
  pinMode(stepPin, OUTPUT);
  // Posible configuración de microstepping si usas DRV8825/A4988
}

void loop() {
  // Mover en una dirección
  digitalWrite(dirPin, HIGH); // O LOW, dependiendo de la dirección
  for(int x = 0; x < stepsPerRevolution; x++) {
    digitalWrite(stepPin, HIGH);
    delayMicroseconds(500); // Velocidad del paso
    digitalWrite(stepPin, LOW);
    delayMicroseconds(500);
  }
  delay(1000); // Esperar un segundo

  // Mover en la dirección opuesta
  digitalWrite(dirPin, LOW);
  for(int x = 0; x < stepsPerRevolution; x++) {
    digitalWrite(stepPin, HIGH);
    delayMicroseconds(500);
    digitalWrite(stepPin, LOW);
    delayMicroseconds(500);
  }
  delay(1000);
}
